# kakeibo-app

家計簿管理アプリケーション。収支の記録・集計・可視化を行う。

## Git 運用ルール

**コードを変更するたびに GitHub へプッシュすること。**

1. 変更をステージング・コミットし、即座に `git push` を実行する
2. コミットメッセージは変更内容を端的に日本語または英語で記述する
3. 破壊的な操作（force push、reset --hard 等）は事前にユーザーの確認を取る
4. `main` ブランチへの force push は禁止

### コミット手順

```
git add <変更ファイル>
git commit -m "変更内容の説明"
git push origin <ブランチ名>
```

## 開発方針

- シンプルさを優先し、不要な抽象化・過剰な機能追加は行わない
- セキュリティ上の問題（XSS、SQL インジェクション等）は即座に修正する
- コメントは「なぜ」が自明でない箇所にのみ記述する
